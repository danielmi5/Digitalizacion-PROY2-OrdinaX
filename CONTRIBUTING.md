# Contribuir al Proyecto

¡Gracias por tu interés en contribuir a este proyecto! Valoramos tus sugerencias, mejoras e informes de errores.  
Por favor, sé respetuoso y mantén una mente abierta en las discusiones y revisiones de código. Trata a los demás como te gustaría ser tratado.  
A continuación, te dejamos algunas pautas para ayudarte a comenzar:

## 💡 Cómo Contribuir

### 1. Haz un fork del repositorio

Haz clic en el botón "Fork" en la parte superior derecha de la página del repositorio y clona tu fork localmente:

```bash
git clone https://github.com/tu-usuario/tu-repositorio.git
cd tu-repositorio
```

### 2. Crea una rama

Crea una nueva rama para tu contribución:

```bash
git checkout -b nombre-de-tu-rama
```

### 3. Realiza tus cambios

Haz tus modificaciones, mejoras o correcciones. Asegúrate de seguir las buenas prácticas del proyecto (estilo de código, estructura de carpetas, etc.).

### 4. Haz commit y push de tus cambios

```bash
git add .
git commit -m "Descripción clara del cambio"
git push origin nombre-de-tu-rama
```

### 5. Crea un Pull Request

Ve al repositorio original y abre un Pull Request desde tu rama.

## 📋 Reglas Básicas

- **Escribe código limpio y legible**, con comentarios cuando sea necesario.
- **Sigue el estilo y la estructura del proyecto.**
- **Incluye pruebas** para cualquier nueva funcionalidad o cambio cuando sea posible.
- **Evita los PR grandes**. Es mejor dividir los cambios en contribuciones pequeñas y enfocadas.

---

## 🔧 Solicitudes de Funcionalidades

Si tienes una idea para una nueva funcionalidad o mejora, no dudes en abrir un *Issue* con:

- Una breve explicación de la funcionalidad.
- Por qué crees que sería útil.
- Cualquier alternativa que hayas considerado.

Esto ayuda a los responsables del proyecto a evaluar y discutir la propuesta antes de comenzar a programar.

## ⚠️ Reporte de Errores

Si encuentras un error, abre un *Issue* con:

- Una descripción clara del problema.
- Pasos para reproducir el error.
- Capturas de pantalla o logs si es aplicable.
